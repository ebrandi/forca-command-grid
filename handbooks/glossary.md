Online concepts it relies on, and the technical and operational vocabulary.

## FORCA Command Grid terms

| Term | Meaning |
|---|---|
| **Command Center** | The pilot's single home dashboard (`/dashboard/`) consolidating readiness, quests, services, and combat rank. |
| **Command Grid** | The application itself — an integrated corporation operations hub. |
| **Quest log / directives** | A member's ranked list of "next best actions" surfaced on the Command Center. |
| **Doctrine** | A named group of standard ship fits published by leadership. |
| **Readiness** | A composite score of how prepared the corporation (or a pilot) is, computed across weighted dimensions. |
| **Dimension** | One weighted input to the readiness index (e.g. doctrine, skill, logistics, financial). |
| **Finding** | A detected gap, risk, or forecast recorded by the readiness platform. |
| **Recommendation** | An explainable, ranked officer suggestion produced by the recommendation engine. |
| **Course of Action (COA)** | A structured proposed action generated by Command Intelligence. |
| **Constraint** | A computed capability ceiling (a binding limit) derived from an intelligence snapshot. |
| **Pingboard** | The unified alerting and calendar subsystem. |
| **Kill feed** | The optional posting of sizeable kills/losses to Discord. |
| **Battle report** | A grouped engagement (system + time window) split into sides. |
| **Combat rank** | A configurable rung on the corp's combat-rank ladder. |
| **Contribution ledger** | The record of activity a pilot did that helped the corp, in native units. |
| **Hall of Fame** | The monthly corp recognition leaderboard. |
| **Member service** | Freight, buyback, or corp store — services with their own audience configuration. |
| **Audience** | A leadership-set visibility level for a feature: disabled, corp, alliance, or public. |
| **Membership gate** | The middleware that confines a logged-in non-member to the recruitment surface. |
| **Admin Console** | The role-gated console at `/ops/` that replaces the stock Django admin. |
| **SRP** | Ship Replacement Program — records and compensates ship losses (never moves ISK itself). |
| **PAP** | Participation record — attendance credit for a fleet operation. |
| **Managed set** | In comms-access sync, the boundary of external roles the app is authorised to manage. |
| **AppSetting** | A key/value store for runtime configuration edited through the console. |

## EVE Online terms

| Term | Meaning |
|---|---|
| **CCP / CCP hf.** | The company that develops and operates EVE Online. |
| **ESI** | EVE Swagger Interface — CCP's official web API for game data. |
| **EVE SSO** | EVE Single Sign-On — CCP's OAuth2 identity service. |
| **SDE** | Static Data Export — CCP's reference data (types, systems, regions, skills). |
| **Scope** | A permission a pilot grants an application to read specific ESI data. |
| **Character / pilot** | An in-game EVE character; a pilot may link several to one account. |
| **Corporation (corp)** | An in-game player organisation; Command Grid serves a single home corp. |
| **Alliance** | A group of corporations. |
| **Director** | An in-game corporation role granting broad corp permissions. |
| **Killmail** | The record of a ship loss, including fit and participants. |
| **Doctrine fit** | A standardised ship loadout. |
| **EFT** | A common text format for ship fittings. |
| **ISK** | InterStellar Kredit — EVE's in-game currency. |
| **Jita** | EVE's main trade hub; the reference market for pricing. |
| **BOM** | Bill of Materials — the inputs needed to build an item. |
| **Ansiblex jump bridge** | A player-deployed structure linking two systems for jump travel. |
| **Cyno beacon** | A structure providing a guaranteed cynosural field for capital jumps. |
| **PI** | Planetary Industry — extracting and processing planetary resources. |
| **Moon extraction** | A scheduled mining operation on a moon, arriving as a "chunk". |
| **Sovereignty / ADM** | Nullsec territory control and its Activity Defense Multiplier. |
| **Upwell structure** | A player-owned station/refinery/engineering complex. |
| **Fuzzwork** | A community service providing SDE conversions and market prices. |
| **EveRef** | A community service providing reference data and historical archives. |
| **zKillboard** | A community killboard whose feed is ingested for corp killmails. |

## Roles and permissions

| Term | Meaning |
|---|---|
| **Public** | Anonymous visitors and logged-in non-members. |
| **Member** | A pilot whose character is in the home corporation. |
| **Officer** | Line leadership tier. |
| **Director** | Senior leadership tier (auto-granted from the in-game Director role). |
| **Admin** | Superuser accounts; bypass gates. |
| **Recruiter** | A lateral capability to work the recruitment pipeline without full officer rank. |
| **FC (fleet commander)** | A lateral capability to manage fleet operations without full officer rank. |
| **Dual control** | Granting the Director role requires a second director's approval. |
| **Classification** | A clearance tier gating Command Intelligence reports (corp-internal → director-eyes-only). |

## Technical terms

| Term | Meaning |
|---|---|
| **Django** | The Python web framework the application is built on. |
| **Celery** | The background task queue; **Beat** is its scheduler. |
| **PostgreSQL** | The system-of-record database. |
| **Redis** | The cache and Celery broker. |
| **gunicorn** | The WSGI application server running Django. |
| **nginx** | The reverse proxy terminating TLS and caching images. |
| **OAuth2 / PKCE** | The authorization-code login flow (with a Proof Key for Code Exchange). |
| **JWT** | A signed token format; EVE SSO access tokens are JWTs, validated server-side. |
| **Fernet** | The symmetric encryption used to protect stored tokens and credentials. |
| **CSP** | Content-Security-Policy — a browser security header; the app uses a per-request nonce. |
| **SSRF** | Server-Side Request Forgery; guarded by outbound host allowlists. |
| **ETag** | An HTTP cache validator the ESI client uses to avoid redundant reads. |
| **Migration** | A versioned database schema change. |
| **htmx / Alpine / Tailwind** | The vendored front-end libraries (partial updates, behaviour, styling). |

## Operational terms

| Term | Meaning |
|---|---|
| **Compose stack** | The set of Docker services (nginx, web, worker, beat, postgres, redis). |
| **Bootstrap** | Loading EVE reference data (SDE, PI rulebook, images, prices) into a fresh instance. |
| **Cache warmer** | A scheduled job that keeps expensive pages precomputed in cache. |
| **Inert / armed** | A feature that is scheduled but does nothing until leadership configures ("arms") it. |
| **Retention** | The policy governing how long historical/operational data is kept. |
| **Health check** | The `/healthz` endpoint and the `scripts/healthcheck.sh` script verifying the stack. |
| **TLS termination** | Decrypting HTTPS at nginx before proxying to gunicorn. |
| **Break-glass** | A deliberate emergency measure (e.g. re-enabling the Django admin). |
